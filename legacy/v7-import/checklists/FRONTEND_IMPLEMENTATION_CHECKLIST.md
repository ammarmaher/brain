# Front-End Implementation Checklist

## Before coding

- [ ] Identify app/lib affected.
- [ ] Inspect existing file structure.
- [ ] Inspect existing components/services/directives.
- [ ] Confirm PrimeNG/PrimeFlex patterns.
- [ ] Confirm theme token pattern.
- [ ] Confirm API/DTO names.
- [ ] Confirm permissions/guards.
- [ ] Confirm loading/empty/error states.

## During coding

- [ ] Use smallest safe change.
- [ ] Keep component logic thin.
- [ ] Use typed models.
- [ ] Reuse existing services.
- [ ] Avoid hardcoded colors/spacing.
- [ ] Avoid unrelated rewrites.
- [ ] Preserve existing tests.
- [ ] Preserve current behavior outside requested change.

## After coding

- [ ] Run available lint/build/test.
- [ ] Check affected apps.
- [ ] Check responsive behavior.
- [ ] Check accessibility/focus states.
- [ ] Check empty/loading/error states.
- [ ] Check business tests.
- [ ] Check regression risks.
- [ ] Summarize changed files.
