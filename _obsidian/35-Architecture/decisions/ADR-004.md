# ADR-004 — Why Stencil for shadow components

> **Status:** accepted · **Reversal cost:** medium · **Decided:** 2026-05-16

## TL;DR

Stencil compiles Web Components with Shadow DOM. Each component is authored once in TSX and produces a `<falcon-X>` custom element (works in any framework or vanilla HTML), an Angular wrapper, and (future) React + Vue wrappers. Stencil v4.20.0 in `libs/falcon-ui-core`.

**Forcing functions:** multi-tenant white-label + end-of-`::ng-deep`. Multi-framework portability without 3× component maintenance.

**Acknowledged costs:** build-pipeline complexity, 4 documented layout traps (per memory `project_falcon_ui_core_layout_traps`).
