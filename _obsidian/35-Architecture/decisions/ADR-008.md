# ADR-008 — Why feature-folder pattern

> **Status:** accepted · **Reversal cost:** low · **Decided:** 2026-05-16

## TL;DR

Every feature folder has the same fixed sub-structure:

```
features/<feature>/
  models/models.ts          (every interface, type, enum, class)
  services/services.ts      (every @Injectable)
  resolvers/resolvers.ts    (every Resolve/ResolveFn, if any)
  directives/directives.ts  (every directive, if any)
```

ONE file per type-folder. Everything of that type is in that file. No exceptions.

**Verification (disk-true 2026-05-16):** `HierarchyPageStateService` = **675 lines** (Agent C's earlier 606 was 6 days stale), `models/models.ts` = 367 lines, `services.ts` = 218 lines. Registered via `providers: [HierarchyPageStateService]` on the route — never `providedIn: 'root'`.

**Acknowledged cost:** large single files (675 lines) can intimidate newcomers — exception clause: split when a single file passes ~500 lines AND has multiple cohesive sub-features.
