# ADR-005 — Why dual-render path

> **Status:** accepted · **Reversal cost:** low (drop one variant) · **Decided:** 2026-05-16

## TL;DR

Every Falcon component ships in TWO variants:
- `<falcon-X>` — **Shadow DOM** (Stencil `shadow: true`), styles via per-component `*.tokens.css`
- `<falcon-X-tw>` — **Light DOM** (Stencil `shadow: false`), styles via global Tailwind utility classes

Sibling folders (not subfolders). The Angular wrapper selects via `[useTailwind]` boolean input (default `true` per the production target). Both variants read from the same `*.tokens.css` SSOT so visual parity is by construction.
