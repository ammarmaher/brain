# ADR-001 — Why Falcon library instead of PrimeNG

> **Status:** accepted · **Reversal cost:** irreversible (PrimeNG fully removed) · **Decided:** 2026-05-16

## TL;DR

Falcon ships its own component library (Stencil + Angular wrappers + Tailwind tokens) instead of consuming PrimeNG. Triggered by bundle bloat (admin-console **2,253 KB → 1,210 KB raw**, **568 KB → 335 KB gz**), theming friction, ::ng-deep proliferation, and the need for cross-framework portability.

**Verification:** 0 PrimeNG imports, 0 PrimeIcons imports, 7 packages physically uninstalled, ESLint flat-block enforces no reintroduction.
