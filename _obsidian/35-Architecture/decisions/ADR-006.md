# ADR-006 — Why Identity Service owns user lifecycle

> **Status:** accepted · **Reversal cost:** medium · **Decided:** 2026-05-16

## TL;DR

Identity Service (`auth.falconhub.space/api/`) fronts Zitadel completely. The frontend talks ONLY to Identity Service via `AuthApiService` + `LoginService`. Owns: login state machine, OTP/MFA, forgot-password, first-login flow, refresh-token rotation, IP allowlist, user-lifecycle CRUD, tenant settings sync.

**Verification:** workspace-wide grep returned ZERO direct Zitadel calls. No `oidc-client-ts`, no `angular-auth-oidc-client`. Only `jwt-decode@^3.1.2` installed.

**Trade-offs accepted:** SPOF on Identity, refresh-token race (D-2026-05-16-14), `auth/logout` not yet wired (D-2026-05-16-11), multi-tab token drift.
