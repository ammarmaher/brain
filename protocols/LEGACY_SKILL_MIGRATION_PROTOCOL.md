# Legacy Skill Migration Protocol

Brain SK v0.1 must not lose older Falcon Brain v7 skills. When a task needs Angular, Tailwind, Falcon frontend, RAGE conversion, PDF/reporting, brand/model orchestration, Gemini QA, or voice notification behavior, check the migrated legacy skill map before assuming the skill does not exist.

## Rules

1. Use v8 skills as the primary router and governance layer.
2. Use migrated v7 skills as specialist execution knowledge when they add detail.
3. Do not delete or ignore old skills unless Ammar explicitly approves.
4. If two skills conflict, follow this priority:
   - Brain SK v0.1 governance protocols
   - Architecture wiki / project rules
   - v7 specialist skill details
   - generic best practice
5. Keep old generated historical outputs as reference only; do not treat them as current source of truth unless explicitly selected.
6. When a legacy skill is useful repeatedly, promote its best rules into a clean v8 protocol or skill and record the source in this migration map.
