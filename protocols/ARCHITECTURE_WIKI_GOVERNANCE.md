# Architecture Wiki Governance

Before creating a feature or touching code, check the architecture wiki.

The wiki may contain:
- architect instructions
- diagrams/images
- coding standards
- routing rules
- module boundaries
- PES/security rules
- component usage rules
- integration guidelines

If wiki changed since last scan, rescan impacted sections.

After implementation, propose or generate wiki updates for:
- new architecture decision
- component capability
- feature flow
- API contract
- validation rule
- gap resolution
- approved pattern

Conflicts between wiki, PRD, backend, code, and screenshots must be reported.
