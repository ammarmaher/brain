## Common-confusion pairs flagged

The following pairs are documented in individual term files under the "Common confusions" section. Each is a candidate for the Brain SK `disambiguation/` folder:

1. **Account ↔ Tenant** — Account is the PRD/business term, Tenant is the Identity-system (Zitadel) term for the same concept.
2. **Customer → Tenant** — "Customer" is a banned synonym for Tenant in platform context (per falcon-wiki Glossary).
3. **User Account → User** — "User Account" is a banned synonym for User when referring to identity.
4. **Permission Group ↔ Role** — In Falcon-PRD vocabulary, "Permission Group" is the bundle; in PES/code, "Role" is sometimes used loosely. Canonical: Role identifies the person's job (sys-admin, account-owner, etc.); Permission Group is the bundle of policies.
5. **Wallet Type ↔ Balance Type** — Both are 2-value enums but orthogonal. Balance Type = User|Node (who holds balance). Wallet Type = Single|Multiple (how many wallets per holder).
6. **CommChannel ↔ Application** — Both are billable service categories. CommChannel = communication-bearer (SMS, WhatsApp, Voice, ...). Application = higher-level app product (e.g. AI, Nabaa). Both share the same Config shape (visibility, pricing, status).
7. **Master Wallet ↔ Wallet** — Master Wallet is the *abstract aggregate* of Active-contract WalletRecords. Other Wallets (Comm/User/Node) are *concrete* rows.
8. **Wallet Record ↔ Wallet** — Wallet is the bucket; WalletRecord is the atomic credit-line inside it, tagged with a Contract ID.
9. **Account Owner ↔ Node Admin** — Both are client-side admin roles. AO has full Main+Sub scope; NA has Sub-only scope.
10. **Falcon User ↔ Client User** — Falcon User = T2 internal staff (System Gateway path). Client User = tenant user (Core Gateway path).
11. **Zitadel ↔ Identity Service** — Zitadel is the OAuth provider. Identity Service is the Falcon service that fronts Zitadel. Frontend NEVER calls Zitadel directly.
12. **`status` (Contract) ↔ `status` (CommChannelConfig) ↔ `status` (User)** — three independent state machines that share the same field name.
