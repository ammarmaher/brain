# User

## English
- **Definition:** A person in Falcon with credentials, a `userType` (Falcon or Client), a `role`, a `status` lifecycle, and a binding to a Node + Tenant. Carries username (unique, immutable), email, phoneNumber, password hash, profile picture, permissionGroupId, and audit timestamps.

## Common confusions
- Banned synonym: **"User Account"** → use **User** (when referring to identity). Source: `falcon-wiki/Glossary.md:67`.
- **User ↔ Account** — Account is the *client company*; User is *one person*. A User belongs to a Node which belongs to an Account.
