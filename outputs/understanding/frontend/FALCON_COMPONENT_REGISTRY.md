# Falcon Component Registry (first-pass)

**Stencil tags inventoried:** 94 (47 Shadow DOM `falcon-X` + 47 Light DOM `falcon-X-tw`).
**Angular wrappers inventoried:** 49 (`falcon-angular-X`) under `libs/falcon-ui-core/src/angular-wrapper/components/` + 7 legacy bespoke Angular components under `libs/falcon/src/shared-ui/lib/components/`.

Rows below are ordered alphabetically by selector and consolidate the Shadow/Light-DOM pair plus the Angular wrapper for each component.

> Conventions used in the table:
> - **SELECTOR** is the user-facing tag — the Angular `falcon-angular-X` selector when one exists, otherwise the raw Stencil tag.
> - **STENCIL-SOURCE-PATH** points to the Shadow DOM `falcon-X` Stencil class. A matching `falcon-X-tw` Light DOM tag exists at the parallel `falcon-X-tw/` folder unless explicitly noted "Shadow only".
> - **INPUTS / OUTPUTS** lists the Stencil `@Prop` / `@Event` set (Angular wrappers mirror these via `@Input` / `@Output` or `CustomEvent` listeners; CVA wrappers also expose Angular Forms binding through `ControlValueAccessor`).
> - **VARIANTS** captures size / state / appearance / variant / mode props.
> - **TOKENS-FILE** points to the component-scoped CSS variable file in `libs/falcon-ui-tokens/src/components/`. "—" means a per-component token file is not declared yet.
> - **DUAL-RENDER** flags whether the component ships both Shadow + Light DOM tags. "Shadow only" means no `-tw` companion exists; "Angular only" means no Stencil core (legacy bespoke component).

## Component count summary

- **47 Stencil component pairs** (Shadow + Light) under `libs/falcon-ui-core/src/components/`.
- **49 Angular wrappers** (`falcon-angular-X`) under `libs/falcon-ui-core/src/angular-wrapper/components/`.

- **46 component-token CSS files** under `libs/falcon-ui-tokens/src/components/`.

## Recurring patterns across components

- **Token-driven visual values** — every component reads from `--falcon-<component>-*` CSS variables, allowing the Studio to mutate tokens at runtime and update Shadow + Light renders identically.
- **`shadow: true` Stencil for the canonical tag**, `shadow: false` Stencil for the `-tw` Light DOM tag. Both wrappers share the same `falconXClasses()` helpers in `libs/falcon-ui-core/src/tailwind/`.
- **CVA on every form-control wrapper** — `ControlValueAccessor` + `NG_VALUE_ACCESSOR` token, supports `[(ngModel)]` and Reactive Forms.
- **OnPush + signals** for internal state; `methods()` not `computed()` so `@Input` ref changes drive re-renders (see source comment in `falcon-input.component.ts:106-108`).
- **Per-instance token override pattern** — consumers add a host class (e.g. `add-client-special-input`) to mutate `--falcon-input-*` tokens for a single element. Demonstrated in `client-information-step.component.html:16-17`.
- **`useTailwind` `@Input` (default `true`)** on every dual-render wrapper — toggles the render path between Shadow (`<falcon-X>`) and Light (`<falcon-X-tw>`).
- **Validation deferred** — uploader, single-uploader, OTP, phone-field, email-field all defer validation to the consumer. The component emits visual state only.

## Missing-capability flags worth tracking

- `falcon-angular-dropdown` — no multi-select (use the separate `falcon-angular-multi-select`); no per-item template slot for live language switch (noted in `user-role-status-step.component.ts` comment as Wave 4 follow-up).
- `falcon-angular-table` — `currentPageReportTemplate` and `paginatorTemplate` strings carry PrimeNG-shaped tokens; templating engine for these is internal.
- `falcon-angular-otp` / `phone-field` / `email-field` / `uploader` / `single-uploader` — validation hooks deferred.
- `falcon-calendar` (Stencil) — single-month grid only; range selection not implemented.
- `falcon-card` — no drag-handle / sortable mode.
- `falcon-avatar` — no avatar-group with fallback chain.
- `falcon-tree-panel` (legacy) — still framework-coupled; awaiting Stencil promotion.
- `falcon-organization-hierarchy-tree-tw` — Light-DOM only; Shadow companion not yet shipped.
- `send-credentials-popup` — still uses `<falcon-angular-dialog>` because `<falcon-angular-popup>` doesn't yet have a slot-friendly variant.
- Legacy components (`falcon-calendar` façade, `falcon-stepper` PrimeNG wrapper, `falcon-multiselect`, `falcon-mobile-number`) — candidates for deprecation once consumers fully migrate.
