# falcon-insufficient-balance-dialog — GAPS & UPGRADES

## Known gaps (open for future polish)

### G1. Inline SVGs instead of glyph font
- 6-dot drag grip, jump/step/info/error chevrons + circles are inline SVG.
- Reason: no `grip-vertical` / `chevrons-up` / `chevrons-down` glyphs in the vendored Falcon icon font yet.
- Path forward: when the library agent adds those glyphs, swap inline SVG for `<falcon-icon name="...">` in Shadow + Tailwind variants (preserves single token-driven sizing).

### G2. React/Vue cross-framework wrappers
- Stencil's `dist-custom-elements` output target auto-emits React proxies; Vue proxy file regenerated by build (99 entries → `libs/falcon-ui-vue/src/index.ts` confirmed in build log).
- Untested in actual React/Vue demo apps. Demo coverage is out of scope per the user's "skip demos in bundle calc" rule.

### G3. Keyboard-only drag-reorder mode
- HTML5 native drag-drop ≠ keyboard accessible.
- The four reorder buttons (jump-top/step-up/step-down/jump-bottom) provide keyboard parity — every position reachable.
- A WAI-ARIA-compliant keyboard-drag mode (Space to grab, arrows to move, Enter to drop) is a future enhancement.

### G4. Focus restore on close
- Self-contained — own backdrop + panel. Focus trap is not yet implemented (relies on natural tab order inside panel).
- Future: trap focus inside `.falcon-ib-dialog__panel` while open, restore to opener on close.

### G5. No localStorage persistence
- Intentional (2026-05-15 decision — server is source of truth).
- If a future use-case wants per-user persistence, the CALLER should cache last-ordered IDs in its own state.

### G6. Animation polish
- Backdrop fade + panel slide on open (200ms ease).
- Reorder is instant — no spring on item swap.
- Future: integrate Falcon design-eng motion preset (`view-transition` or spring) on `move()`.
