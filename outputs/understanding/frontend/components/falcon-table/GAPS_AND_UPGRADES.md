# falcon-table — GAPS AND UPGRADES

## Known incomplete capabilities

1. **Per-column filtering** — only a global filter strip is implemented. No per-column filter dropdown / range / typeahead. PrimeNG's per-column filter is a known gap.
2. **Row expansion / details row** — no built-in expand-to-row-details pattern. Use `falcon-angular-tree-table` if the data is hierarchical, otherwise wire a second row manually.
3. **No row drag-and-drop / reordering**.
4. **No row grouping with collapsible group headers** (just `responsiveLayout: 'stack'` which is a different feature).
5. **No CSV / Excel export**. Was provided by PrimeNG; consumers now must roll their own.
6. **No state persistence** — sort / filter / page state is in-memory only. No `stateStorage: 'session'|'local'` like PrimeNG `<p-table>`.
7. **No keyboard cell navigation** — Arrow keys do not move focus between cells; only row-level focus is tracked.
8. **Column resize** — not implemented. Widths come from `column.width` / `column.maxWidth` and CSS only.
9. **Column reorder** — not implemented.
10. **Custom paginator template tokens** — `paginatorTemplate` accepts a fixed set of PrimeNG-shaped tokens (`FirstPageLink`, `PrevPageLink`, `JumpToPageInput`, etc.). Unknown tokens silently no-op.
11. **`rowStyleClass` is per-row only** — no per-cell class callback. Use `column.cellClass` (static string) instead.
12. **`column.render: (row, col) => string`** returns a plain HTML/text string. Not safe-HTML-sanitised. Consumers must escape user content themselves OR use a custom cell template via the data-table wrapper.
13. **No `falcon-angular-table-tw`-only feature parity** with `falcon-angular-data-table` — if you need custom Angular cell templates, you MUST use `<falcon-angular-data-table>` (the Strategy E projection wrapper), not `<falcon-angular-table>`. The two wrappers are not fully interchangeable.

## a11y status

- `role="grid"` + `aria-rowcount` + per-`<th>` `aria-sort` + per-`<tr>` `aria-selected` + `aria-busy` on tbody during loading.
- Empty-state cell renders inside a `<td>` with `aria-live="polite"` so screen readers announce "No records to display" when data clears.
- Sort buttons inside headers have `aria-label` like `"Sort by Name ascending"`.
- Row checkboxes have `aria-label="Select row {id}"`.
- TODO: Confirm `aria-rowindex` per `<tr>` for virtualisation scenarios.
- TODO: ARIA live-region announcement when `globalFilterValue` filters out rows.

## Test coverage

- No `.spec.ts` / `.e2e.ts` adjacent to the source. Coverage gap.

## Performance notes

- The Stencil component re-renders fully on `rows` / `columns` / `selectedRowIds` / `sortBy` ref change. No row-level memoisation.
- Sort is in-process (in the component) when `lazy=false`. Switch to `lazy=true` for >5,000-row datasets.
- Skeleton-loading body renders `skeletonRows` placeholder rows (default 6) — bound by token-driven height, not real row content.
- No virtualisation. For >1,000 rows on a single page, consider `lazy=true` + smaller `pageSize`.

## Migration notes (PrimeNG → Falcon)

- `<p-table>` → `<falcon-angular-data-table>` (Strategy E projection) OR `<falcon-angular-table>` (simpler shape).
- `<p-tableHeaderCheckbox>` / `<p-tableCheckbox>` → `selectionMode="multiple"` does this automatically.
- `<ng-template pTemplate="body" let-row>` → `<ng-template [falconCell]="<field>" let-row>` (data-table only).
- `<p-tablePaginator>` → `[paginator]="true"` + `[totalRecords]` + `[pageSize]` + `[rowsPerPageOptions]`.
- `loading` + `loadingIcon` → `[loading]="true"` (loading icon is part of the skeleton body).
- `lazy` + `(onLazyLoad)` → `[lazy]="true"` + `(lazyLoad)`.
- `globalFilterFields` + `(filter)` → `[globalFilterFields]` + `[globalFilterValue]` (two-way) + `(falcon-global-filter-change)`.
- `dataKey` → `[dataKey]="'id'"`.
- `frozenColumns` → set `column.frozen: 'left' | 'right'` on each column definition.

## Cross-framework drift checks

- React wrapper auto-generated by Stencil. Should track 1:1.
- Vue wrapper script — verify after API changes.
- The Strategy E projection pattern (custom Angular cell templates) is **Angular-only**. React + Vue consumers use `column.render` callbacks or compose around the Stencil tag directly.

## Upgrade ideas (out of scope here; capture only)

- Add per-column filter API: `column.filter: { type: 'text' | 'dropdown' | 'date' | 'number' | 'custom' }` + a popover trigger in the header.
- Add row expansion: `expandable: boolean` + `expansionRowTemplate` slot.
- Add row drag-and-drop with a `dragHandle` column type.
- Add row grouping with a `groupBy` prop + per-group template slot.
- Add state persistence: `stateStorage: 'session' | 'local' | none` + `stateKey`.
- Add `column.render: (row, col) => string` safe-HTML opt-in via `[trustHtml]="true"` Angular wrapper input.
- Add column resize + reorder behind opt-in flags.
- Wire `aria-rowindex` per `<tr>` for virtualisation.
- Add `.spec.ts` + `.e2e.ts` covering at minimum: sort single + multi, selection single + multi, lazy mode, global filter, skeleton-loading, empty state, frozen + sticky.
- Consider sharing a `<falcon-angular-table>` that also accepts `[falconCell]` templates so the data-table / table split becomes a single component.
