P3   C1 — Order-sensitive dirty-tracking

Code shot — template-management.mappers.ts:52
/** Stable string snapshot for dirty-tracking. */ export function serializeChannels(channels: ChannelLocalState[]): string { return JSON.stringify(channels); // ← order-sensitive } // consumed by template-form.service.ts:90 get isDirty(): boolean { return serializeChannels(this.channels) !== this.pristineSnapshot; }
What is wrong

JSON.stringify preserves array order. level1UserIds / level2UserIds are string[] populated from a multiselect. Selecting the same users in a different order changes the string → isDirty returns true with no real change → the user is wrongly warned about unsaved changes. (False positives only — never a missed change — but still a real UX defect.)

Suggested fix
/** Order-insensitive snapshot for dirty-tracking. */ export function serializeChannels(channels: ChannelLocalState[]): string { const norm = channels .map((c) => ({ ...c, level1UserIds: [...c.level1UserIds].sort(), level2UserIds: [...c.level2UserIds].sort(), })) .sort((a, b) => a.channelId.localeCompare(b.channelId)); return JSON.stringify(norm); }

Nothing functionally broken — the error is caught and an empty list degrades cleanly. It is flagged only because raw console.error bypasses any central logging/telemetry. Acceptable to ship; listed for completeness.
